DATE    : 2026-04-01
PROMPT  : "The root cause is configparser.ConfigParser() uses BasicInterpolation
           by default — it scans config values for %(key)s patterns. The Unicode
           box-drawing characters in the decorative comment immediately after
           max_lines = 2 (no blank line separator) could trip the parser into
           misreading the section context. Fix: switch to RawConfigParser to
           disable interpolation entirely, and add a blank line after max_lines
           in config.ini."
FILES   : supertitle.py, config.ini
CHANGES :
  - Changed configparser.ConfigParser() to configparser.RawConfigParser() in
    load_config() to disable BasicInterpolation
  - Added blank line after max_lines value in config.ini to cleanly separate
    the value from the decorative comment block that follows it
